```markdown
# 需求：熄灯打卡 - 睡眠时间追踪工具

## 项目概述
创建一个单文件 HTML 应用（含 CSS/JS），用于记录每天熄灯睡觉的时间，帮助用户追踪睡眠习惯。数据存储在 localStorage，支持导入导出。

## 技术要求
- 单个 HTML 文件，内联 CSS 和 JS
- 移动端优先，响应式设计
- 深色主题（护眼）
- 数据存储：localStorage
- 不依赖任何外部框架（可用 Google Fonts）

---

## 功能需求

### 1. 打卡记录
- 点击按钮记录当前时间为熄灯时间
- **关键逻辑**：8:00 前打卡算作前一天的记录
  - 例：04/14 01:20 打卡 → 归属 04/13
  - 例：04/14 08:00 打卡 → 归属 04/14
- 同一天重复打卡则覆盖（以最后一次为准）
- 显示今日打卡状态

### 2. 补录/修改
- 点击日历中的某一天，弹窗可手动输入时间
- 支持补录过去的记录
- 支持修改已有记录
- 补录/修改的记录标记 `editedAt` 时间戳

### 3. 日历视图
- 显示当月每天的打卡时间
- 根据时间阈值用颜色区分：
  - 🟢 绿色：早睡（早于设定时间）
  - 🟡 黄色：一般（介于两者之间）
  - 🔴 红色：熬夜（晚于设定时间）
- 支持切换上/下月
- 今天的日期有特殊边框标识

### 4. 时间阈值设置
- 用户可设置两个时间阈值：
  - 早睡时间（默认 23:00）
  - 熬夜时间（默认 00:30）
- 设置后自动保存

### 5. 趋势曲线
- 折线图显示入睡时间变化趋势
- 支持切换时间范围：7天 / 14天 / 30天 / 全部
- Y轴为时间（如 22:00 - 03:00）
- X轴为日期

### 6. 统计卡片
- 支持切换维度：本周 / 本月 / 年度
- 显示内容：
  - 早睡天数
  - 熬夜天数
  - 平均入睡时间
  - （年度额外显示）总打卡天数

### 7. 数据管理
- **导出 JSON**：下载完整数据备份
- **导出 CSV**：下载表格格式（日期,时间），Excel 可打开
- **导入 JSON**：从文件恢复数据
- **清除数据**：删除所有记录，需二次确认弹窗

## 数据结构

```json
{
  "settings": {
    "earlyTime": "23:00",
    "lateTime": "00:30"
  },
  "records": [
    { "date": "2026-04-10", "time": "23:45", "editedAt": null },
    { "date": "2026-04-11", "time": "01:20", "editedAt": "2026-04-12T10:30:00" }
  ]
}
```

- `date`：归属日期（YYYY-MM-DD）
- `time`：熄灯时间（HH:MM，24小时制）
- `editedAt`：如果是补录/修改，记录编辑时间；否则为 null

## 交互细节

1. **打卡按钮**
   - 点击后立即记录，显示 toast 提示"打卡成功 23:45"
   - 如果今天已打卡，按钮文字可变为"更新打卡"

2. **日历点击**
   - 点击任意日期弹出 modal
   - modal 内容：日期 + 时间输入框 + 保存/取消/删除按钮
   - 如果该天有记录，显示现有时间；否则为空

3. **清除数据**
   - 点击后弹出确认框："确定要清除所有数据吗？此操作不可恢复"
   - 确认后清空，显示 toast "数据已清除"

4. **导入数据**
   - 选择 JSON 文件后自动导入
   - 导入成功显示 toast "已导入 X 条记录"
   - 如果格式错误，提示"文件格式错误"
